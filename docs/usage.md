# 使用说明

本文档面向实际打麻将时的手机端使用场景。

## 访问应用

本地开发环境访问：

```text
http://localhost:8899
```

NAS Docker 部署后访问：

```text
http://NAS局域网IP:8899
```

同一张麻将桌上的玩家需要连接同一个局域网，并访问同一个地址。

## 创建房间

1. 打开首页。
2. 在“创建房间”区域选择三人或四人模式。
3. 填写房间名称和房主昵称。
4. 设置起始分、计分单位、是否允许负分。
5. 点击“创建房间”。

创建成功后，页面会进入房间等待页，并显示 6 位房间号。

## 加入房间

1. 打开首页。
2. 在“加入房间”区域输入房间号。
3. 输入玩家昵称。
4. 点击“加入房间”。

昵称在同一个房间内不能重复。项目当前没有账号系统，玩家身份保存在当前浏览器的本地存储中。

## 开始对局

房间处于等待状态时，房主可以点击“开始对局”。

后端允许至少 2 名玩家开始，用于测试和灵活使用。正式打牌时建议等三人或四人到齐后再开始。

## 记分

对局开始后，页面顶部会显示所有玩家当前分数。

记录一笔分数：

1. 选择付款方。
2. 选择收款方。
3. 输入分数，或点击快捷分数按钮累加分数。
4. 可选填写备注，例如“点炮”“杠分”“其他”。
5. 点击“确认记分”。

提交后，后端会：

- 写入流水；
- 扣减付款方分数；
- 增加收款方分数；
- 向房间内所有客户端广播最新状态。

## 撤销流水

流水列表中的每一笔有效流水都可以申请撤销。只有该笔流水的付款方和收款方都确认后，系统才会真正撤销并回滚分数。

撤销不会删除原始流水，而是把流水标记为已撤销，并记录收付双方确认状态。这样可以保留对账痕迹，也避免单方误撤销。

## 结束结算

房主点击“结束结算”后，对局状态变为已结束，并生成每位玩家的结算结果：

- 起始分；
- 最终分；
- 输赢分；
- 按计分单位换算后的结果。

所有玩家的输赢分总和应为 0。

## 历史记录

首页右上角可以进入“历史”。

历史记录展示已结束对局，点击某一局可以查看：

- 玩家结算；
- 完整流水；
- 已撤销流水标记。

## 使用限制

- 不计算麻将番型。
- 不判断胡牌、自摸、点炮等麻将规则。
- 没有用户登录和跨设备身份恢复。
- 默认用于局域网娱乐积分记录，不建议直接暴露到公网。
