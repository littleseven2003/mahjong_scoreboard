# 使用说明

本文档面向实际打麻将时的手机端使用场景。

## 访问应用

本地开发环境访问：

```text
http://localhost:8899
```

NAS Docker 部署后访问：

```text
http://NAS局域网IP:8899
```

同一张麻将桌上的玩家需要连接同一个局域网，并访问同一个地址。

## 创建房间

1. 打开首页。
2. 点击“创建房间”进入创建页。
3. 选择三人或四人模式。
4. 填写房间名称和房主昵称。
5. 设置起始分、计分单位、是否允许负分。
6. 点击“创建房间”。

创建成功后，页面会进入房间等待页，并显示 6 位房间号。

## 继续对局

首页会显示当前浏览器加入过、且尚未结束的对局。

未完成对局从创建或开始后超过 12 小时，会在继续对局列表中标记为“已超时”。超时项会显示“删除”按钮，用于删除本机保存的继续入口。

删除只清理当前浏览器的入口记录，不会删除服务器上的房间或历史数据。

## 加入房间

1. 打开首页。
2. 点击“加入房间”进入加入页。
3. 输入房间号。
4. 输入玩家昵称。
5. 点击“加入房间”。

昵称在同一个房间内不能重复。项目当前没有账号系统，玩家身份保存在当前浏览器的本地存储中。

玩家在等待阶段退出后，该昵称会被释放，其他设备可以使用同一个昵称重新加入。

## 准备与退出

等待阶段中，房主不需要准备。

非房主进入房间后可以：

- 点击“准备”；
- 点击“取消准备”；
- 点击“退出房间”。

当房间人数已满，且除房主外的所有玩家都准备完毕后，房主会收到提示。

房主可以在等待阶段随时解散房间。房间解散后，房间内所有玩家都会收到“房间已解散”的提示，并回到首页。

## 开始对局

房间处于等待状态，人数已满，且所有非房主玩家都已准备时，房主可以点击“开始对局”。

后端要求房间人数满足创建时选择的三人或四人模式，并且所有非房主玩家都已准备，才能开始对局。

## 记分

对局开始后，页面顶部会显示所有玩家当前分数。

记录一笔分数：

1. 选择扣分玩家。
2. 选择加分玩家。
3. 输入分数，或点击快捷分数按钮累加分数。
4. 可选填写备注，例如“点炮”“杠分”“其他”。
5. 点击“确认记分”。

非房主只能以自己作为扣分玩家。扣分玩家与加分玩家不能相同，选择一方后，另一方列表会自动排除已选玩家。

提交后，后端会：

- 写入流水；
- 扣减扣分玩家分数；
- 增加加分玩家分数；
- 向房间内所有客户端广播最新状态。

## 撤销流水

流水列表中的每一笔有效流水都可以申请撤销。只有该笔流水的扣分玩家和加分玩家都确认后，系统才会真正撤销并回滚分数。

撤销不会删除原始流水，而是把流水标记为已撤销，并记录双方确认状态。这样可以保留对账痕迹，也避免单方误撤销。

当一方发起撤销后，另一方会看到撤销确认弹窗，可以选择：

- 确认撤销：同意撤销；
- 取消撤销：拒绝本次撤销申请；
- 暂时关闭：先关闭弹窗，流水栏中仍保留处理按钮。

如果另一方取消撤销，本次撤销申请会失效，需要重新申请。

## 结束结算

房主点击“结束结算”后，对局状态变为已结束，并生成每位玩家的结算结果：

- 起始分；
- 最终分；
- 输赢分；
- 按计分单位换算后的结果。

所有玩家的输赢分总和应为 0。

## 历史记录

首页右上角可以进入“历史”。

历史记录展示已结束对局，点击某一局可以查看：

- 玩家结算；
- 完整流水；
- 已撤销流水标记。

## 使用限制

- 不计算麻将番型。
- 不判断胡牌、自摸、点炮等麻将规则。
- 没有用户登录和跨设备身份恢复。
- 默认用于局域网娱乐积分记录，不建议直接暴露到公网。
